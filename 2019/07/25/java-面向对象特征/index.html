java(面向对象特征)

发表于 2019-07-25

1)向对象的特征是封装，继承，多态
封装设计是为了防止直接访问更改字段造成程序错误
封装屏蔽了程序内部实现细节，提高数据的安全性,简化操作
继承是使用已有类派生出子类
继承可以实现代码的复用
多态是使用相同的代码实现不同操作
多态可以提高代码的可扩展性
2)封装
1.访问权限
(1)类的访问权限
类访问权限有两个: 公共类与非公共类
使用public修饰的类就是公共类,可以在其他包中使用,类名与源文件名相同
没有public修饰的类就是非公共类, 只能在当前包中使用
(2)类成员的访问权限
公共的(public)可以在任何位置使用
受保护的(protected)在当前类与子类中使用
默认的可以在当前包中使用
私有的(private)只可以在当前类中使用
注意 权限的使用遵循最小权限原则
2.封装就是使用private把字段修饰为私有的，提供getter/setter实现对字段的访问, 必要时在setter方法中对数据进行验证

3)继承
继承就是使用父类派生子类
继承需要满足is a关系
1. 子类 extends 父类,子类继承了父类就自动拥有了父类的实例变量与实例方法
2. 静态成员没有继承问题
3. 构造方法不能被继承, 子类创建对象调用子类构造方法, 在子类构造方法体执行前会先调用父类的构造方法, 默认情况下调用父类的无参构造, 如果父类没有无参构造系统报错, 在子类构造方法体中也可以通过super()显示调用父类的构造方法
4. 子类从父类继承的方法如果不能满足子类需求,在子类 中可以重写该方法, 在子类的类体中通过super调用父类的方法
5.super关键字的作用
(1)在子类构造方法中通过super()显示调用父类的构造方法
(2)在子类的类体中通过super表示该成员是从父类继承来的
注意:
super跟父类对象没有关系
super()调用父类构造方法的语句必须作为第一条语句存在, super()与this()不能共存

4)多态
多态分为对象多态与行为多态
对象多态是指同一个对象具有不同的状态,是通过对象的转型实现的
行为多态就是相同的语句执行不同的操作,或者调用相同的方法执行不同的方法体.
行为多态又分为静态多态与动态多态.
静态多态是通过方法重载实现的,在调用某个方法时,传递不同的参数执行不同的方法体,在编译(javac)阶段就可以确定要执行的代码,又称为编译时多态
动态多态是在程序运行到这一行时才能确定要执行哪个对象的方法,又称为动态绑定,动态时多态
如果不特殊说明,平时所说的多态就是运行时多态
运行时多态实现有四个条件:
1. 子类继承父类
2. 子类重写父类的方法
3. 给父类的引用赋值子类对象(对象向上转型)
4. 通过父类引用调用被重写方法时,实际执行的是子类 对象的方法
对象向下转型
当父类引用指向子类对象时, 不能通过父类引用访问子类特有的属性与方法
如果需要访问子类特有的成员时需要对象向下转型
对象向下转型通过强制类型转换实现
一般情况下在对象向下转型前会使用instanceof运算符判断堆中对象是否为目标类型
5.多态的作用:
(1) 体现了面向抽象编程的思想
(2) 降低了程序的耦合度
(3) 提高了程序的可扩展性
