java (构造方法与static关键字)

发表于 2019-07-24

构造方法
1)作用:
在创建对象的同时,给对象的各个属性赋值(初始化)
2)特点:
构造方法名与类名相同
构造方法没有返回值,也没有返回值类型
定义了类之后 ,系统在生成一个默认无参构造方法(缺省构造器)
当显示的定义了构造方法之后,系统默认的无参构造就不存在了
在开发中,经常 使用无参构造方法创建对象, 可以把无参构造方法显示的定义出来,实现构造方法的重载
注意构造方法只能在创建对象时, 由系统调用, 不能通过类名/对象名调用构造方法, 创建对象new运算符后面跟的实际上是构造方法

2)static关键字,可以修饰成员变量, 修饰成员方法,修饰代码块,修饰静态内部类,实现静态导入
1.静态变量
1) 在类体中定义的变量称为成员变量
使用static修饰的成员变量称为静态变量
没有使用static修饰的成员变量称为实例变量
2)静态变量特点:
(1)在类加载内存时就给静态变量分配存储空间, 一直到程序结束
(2)静态变量存储在方法区, 实例变量存储在堆区, 局部变量存储在栈区
(3)静态变量是整个类所有对象共有, 不属于某个具体对象, 一般通过类名直接访问
3)在什么情况下会定义静态变量?
(1)当所有对象需要共享同一个数据时,可以把共享的数据定义为静态变量
(2)当所有对象的某个属性值都一样时,可以定义为静态变量
2.静态方法
1)在类体中定义的方法称为成员方法
如果没有static修饰的成员方法称为实例方法
使用static修饰的成员方法称为静态方法
2)静态方法特点:
在静态方法中,不能 直接使用非静态的成员, 不可以使用this关键字
在实例方法中可以直接使用实例变量,也可直接使用静态成员
3)应用场景:
一般情况下,在设计工具类时把方法定义为静态的,可以直接通过类名调用方法, 使用比较方便
如:
Arrays.sort()
Arrays.copyOf()
Arrays.toString()
3.静态代码块
static {
System.out.println(“这是静态代码块, 在类加载内存时执行静态代码块, 经常在静态代码块中给静态变量初始化, 也可以在静态代码块中加载类依赖的外部资源, JDBC驱动的加载”);
}

java(接口)

发表于 2019-07-24

1)接口的概念
接口是功能的封装，主要用来描述类具有怎样的功能的
2)接口的定义
修饰符 interface 借口名{
功能使用方法定义，默认使用public abstact修饰
}
3)接口的使用
修饰符 类名 implement 接口名{
类实现了接口，需要重写接口的抽象方法
}
说明
1) 接口可以简单的看作是功能的封装, 接口中功能使用方法表示,接口中的方法默认public abstract修饰
2) 类实现了接口需要重写接口中的抽象方法,如果没有重写所有抽象方法,则类也需要定义为抽象类
3) 接口是一种引用数据类型,但是接口不能实例化对象,接口引用可以赋值实现类对象或者赋值匿名内部类对象,
4) 通过接口引用调用抽象方法时实际执行的是实现类对象的方法,称为接口多态
5) 在JDK8中,接口包含四部分内容: 默认的抽象方法, 字段默认使用public static final修饰为常量 ,可以添加static静态方法, default修饰的方法
6) 接口允许继承,并且允许多继承
7) 一个类可以同时实现多个接口 ,需要重写所有接口中的抽象方法
4)接口的应用:
1) 接口用来封装某个功能, 如Flyable接口封装了飞行功能
2) 接口定义一组操作规范,一个协议, 如把对集合的一组操作封装了Collection接口中
3) 接口在某种程度上可以解决类的多继承
5)面向接口的编程
1) 接口比较灵活,一个类在继承父类的同时可以实现多个接口
2) 体现面向抽象编程的思想,提高程序的可扩展性,降低类的耦合度
3) 通过接口可以使项目分层.

java (集合 2 map)

Map
Map集合是按<键,值>对的形式存储数据
1)map集合的常用方法
void clear() 清空集合中所有的键值对.
boolean containsKey(Object key) 判断当前map中的键是否包含指定的key
boolean containsValue(Object value) 判断当前map中的值是否包含指定的value
Set<Map.Entry<K,V>> entrySet() 返回所有Entry的集合.
V get(Object key) 返回键key对应的值
Set keySet() 返回所有键的集合
V put(K key, V value) 向map中添加<key,value>键值对
void putAll(Map<? extends K,? extends V> m)
V remove(Object key) 从map中删除key匹配的键值对
default boolean remove(Object key, Object value)
default V replace(K key, V value) 在map中把key的值替换为value
int size() 返回键值对的数量
Collection values() 返回所有值的集合
2)Map集合常用实现类
主要有HashMap，Heshable，TreeMap
1.HashMap:
底层实现是哈希表，哈希表是一个数组，数组每个元素都是单向链表
工作原理
首先根据键的哈希码经过哈希函数计算哈希值
然后根据哈希值计算对应数组下标
最后访问对应数组下表的链表
HashMap是如何解决哈希冲突(碰撞)?
采用链表法.
哈希冲突是指不同的键计算出数组的下标相同
不同键的哈希码可能相同;
不同的哈希码经过hash函数计算的hash值可能相同
不同的hash值计算出的数组下标可能相同
JDK8对HashMap做了哪些改进?
1) 把新增的结点插入到链表的尾部. 在JDK7之前新增的结点插入到链表的头部
2) 当单向链表中结点的数量超过8个时, 系统会自动的把单向链表转换为红黑树结构
2.hashable
HashMap与HashTable的区别
1) 底层都是哈希表, HashTable是线程安全的, HashMap不是线程安全的
2) HashMap的初始化容量: 16, HashTable的初始化容量:11
3) 加载因子: 0.75, 当键值对数量 大于 加载因子*容量,扩容
4) HashMap扩容:2倍, HashTable扩容:2倍 + 1
5) HashMap的键与值都可以是null, HashTable的键值都不能为null
6) 在创建集合时可以指定初始化容量, HashMap会自动的调整为2的幂次方, 为了快速计算数组的下标; HashTable不调整
3.Treemap
TreeMap实现了SortedMap接口,可以根据键自然排序,要求键必须是可比较的
1) 在构造方法中指定Comparator比较器
2) 如果构造方法中没有Comparator比较器,要求键实现Comparable接口
3)小结
1.HashMap
底层是哈希表
默认初始化容量: 16
加载因子: 0.75
扩容: 2倍
键与值都可以为null
可以指定初始化容量, 系统会自动的调整为2的幂次方
2.HashTable
底层是哈希表, 它是 线程安全的, HashMap不是线程安全的
默认初始化容量: 11
加载因子: 0.75
扩容: 2 倍 + 1
键值都不能为null
可以指定初始化容量, 不调整
3.Properties
键值都 是String
经常用于设置/读取系统属性
setProperty(属性名, 属性值)
getProperty(属性名)
4.TreeMap
实现了SortedMap接口, 可以根据键自然排序, 要求键必须是可比较的
1) 在构造方法中指定Comparator
2) 如果构造方法没有Comparator,要求键实现Comparable接口
TreeMap的键排序是根据红黑树原理排序
注意集合只能存储引用类型，存基本类型是存对应包装类

java中的集合主分为两类Collection和map集合
Collection集合
Collection接口从父接口iterable接口iterator()方法
该方法返回一个迭代器，所以所有实现Collection 接口的实现类都要重写iterator方法
1)Collection基本操作
boolean add(E e) 把元素e添加到当前集合中
boolean addAll(Collection<? extends E> c) 把参数c集合中所有的元素都添加到当前集合中
void clear() 清空集合所有的元素
boolean contains(Object o) 判断当前集合中是否包含元素o
boolean isEmpty() 判断集合是否为空.
Iterator
iterator() 返回集合的迭代器
boolean remove(Object o) 从当前集合中删除第一个与o匹配的元素
int size() 返回集合中元素的数量
Object[]
toArray() 把集合转换为数组
2)Collection
子接口有list接口和set接口
1.
-List集合
特点:有序,可重复
存储顺序与添加顺序 一样, 可以存储重复的数据
List为每个元素指定了索引值,增加了针对索引值的操作:
add( index, o)
remove(index)
get(index)
在JDK8中增加了sort(Comparator)排序功能
list集合实现类经常使用的有ArrayList，Vector，linkedlist：
(1)Arrayslist
底层是数组, 访问快,添加删除慢
初始化容量: 10
扩容:1.5倍
(2)Vector
底层是数组, 线程安全的,ArrayList不是线程安全的
初始化容量:10
扩容: 2倍
(3)LinkedList
底层是双向链表, 访问慢,添加/删除效率高
增加了针对第一个/最后一个元素的操作
模拟栈, 模拟队列
应用选择
如果存储可以重复的数据选择List集合
如果以访问为主,选择ArrayLIst
如果频繁的进行添加删除操作选择LinkedList
注意：List集合在contains(o), remove(o)方法中,需要调用对象的equals()方法,如果存储自定义类型的元素需要重写equals()/hashCode()
2.
Set
存储特点:无序不可重复
存储顺序与添加顺序可能不一样, 不能存储重复的数据
(1)HashSet
底层是HashMap
HashSet就是HashMap键的集合
(2)TreeSet
底层TreeMap
TreeSet就是TreeMap键的集合
实现了SortedSet接口,可以对集合中的元素自然排序,要求元素必须是可比较的
1) 要么在构造方法中指定Comparator比较器
2) 如果构造方法没有指定Comparator,要求元素的类实现Comparable接口
应用场景
如果存储不重复的数据选择Set
如果不需要排序就使用HashSet
如果需要排序就使用TreeSet
注意：
HashSet判断是否相同的元素需要调用equals()方法,如果存储自定义类型数据,需要重写equals()/hashCode()
TreeSet判断是否相同的元素,是根据Comparator/Comparable的比较结果是否为0来判断的,如果比较结果为0就认为是相同的元素

java回忆（类与对象）

发表于 2018-03-08 更新于 2019-07-24

1）java是一种面向对象的编程语言
面向对象程序设计就是把问题分解为若干的对象,分析每个对象具有哪些特征与行为;
面向对象程序设计就是以对象为中心,通过给对象发送消息(调用方法)解决问题
面向对象程序设计与面向过程程序设计相比,更符合人的自然思维方式
面向对象程序设计特点:封装,继承,多态
2)类与对象
对象是具体的事物，世间万物都是对象
类是一组对象的相同的特征与行为抽象而成的
类是抽象的, 对象是具体的
类可以看作一个模板, 对象就是类的一个实例( Instance )
在java中需要先创建类，再创建对象。创建对象的过程称为对象的实例化
3)类的创建
语法格式
修饰符 class 类名{
特征使用属性表示
行为使用方法表示
}
1.)属性保存在变量中, 在类体中定义的变量也叫成员变量,在方法体中定义的变量称为局部变量
2.)行为(操作) 使用方法表示, 在类体中定义的方法称为成员方法
4)对象的创建
对象通过new运算符创建
new 运算符将在堆中开辟内存空间存放对象
创建了对象,系统会给对象的属性默认初始化,如果是数据类型默认初始化为0, 布尔类型默认初始化为false, char字符类型默认初始化为码值为0的字符’\u0000’, 引用类型属性默认初始化为null。
语法格式
类名 对象名 = new 类名（）；
对象名就是一个变量名,对象名中保存堆中对象的引用(起始地址)
堆中new出来的才是真正的对象, 对象名准确的说是对象的引用
